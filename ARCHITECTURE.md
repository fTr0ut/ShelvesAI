## 5. Database Schema

**Engine:** PostgreSQL 16 with extensions `uuid-ossp` and `pg_trgm`.

**Schema management:** Knex migrations in `api/database/migrations/` (run `npx knex migrate:latest`).
Baseline schema also in `api/database/init/01-schema.sql` (used by Docker init).

### Core Tables

| Table | PK | Description |
|---|---|---|
| `users` | UUID | Accounts; has `is_admin`, `is_suspended`, `is_premium`, `is_private` flags |
| `profile_media` | SERIAL | Avatar storage (circular FK with users) |
| `shelves` | SERIAL | User shelves; `type` ∈ books/movies/games/vinyl/other; `visibility` ∈ private/friends/public |
| `collectables` | SERIAL | Global catalog; `kind` ∈ book/movie/game/album; fingerprint + lightweight_fingerprint for dedup; `pg_trgm` GIN index on title |
| `editions` | SERIAL | Variants of a collectable |
| `media` | SERIAL | Cached cover images linked to collectables |
| `user_collections` | SERIAL | Items on shelves; FK to either `collectables` OR `user_manuals` (CHECK constraint) |
| `user_manuals` | SERIAL | Custom entries not in catalog |
| `user_ratings` | SERIAL | Decoupled ratings (collectable or manual) |
| `needs_review` | SERIAL | Vision scan results below confidence threshold; `status` ∈ pending/completed/dismissed |
| `friendships` | SERIAL | `status` ∈ pending/accepted/blocked; self-friendship prevented by CHECK |
| `event_aggregates` | UUID | Feed batches; groups events within a time window |
| `event_logs` | SERIAL | Individual feed events; FK to aggregate |
| `event_likes` | SERIAL | Likes on feed aggregates |
| `event_comments` | SERIAL | Comments on feed aggregates |
| `notifications` | UUID | In-app notifications; `type` ∈ like/comment/friend_request/friend_accept |
| `push_device_tokens` | SERIAL | Expo push tokens per device |
| `notification_preferences` | UUID (PK=user_id) | Per-user push opt-in settings |
| `password_reset_tokens` | SERIAL | Time-limited reset tokens |
| `user_vision_quota` | UUID (PK=user_id) | Monthly scan quota tracking |
| `admin_action_logs` | SERIAL | Audit log for admin actions |
| `news_items` | SERIAL | Trending/upcoming items from external APIs; expires_at for cache invalidation |
| `user_news_seen` | SERIAL | Tracks which news items a user has seen |
| `user_news_dismissed` | SERIAL | Tracks dismissed news items |
| `user_favorites` | SERIAL | Favorited collectables |
| `user_lists` | SERIAL | Custom ranked lists (max 10 items, position 1–10) |
| `user_list_items` | SERIAL | Items in a list |
| `wishlists` | SERIAL | User wishlists |
| `wishlist_items` | SERIAL | Items in a wishlist (collectable or free-text) |

### Row Level Security (RLS)
Applied via migrations `20260128000000` through `20260128040000`:
- **Tier 1** — Simple user isolation (user_collections, user_manuals, user_ratings, needs_review, etc.)
- **Tier 2** — Visibility-based (shelves, profiles)
- **Tier 3** — Complex (friendships, feed)
- **Tier 4** — Cascading
- Admin bypass policy on all RLS tables via `is_current_user_admin()` DB function.
- RLS context set via `SET LOCAL "app.current_user_id" = $1` in `queryWithContext()` / `transactionWithContext()`.

### Migration Timeline (40 files)
Migrations span `20260110` → `20260128`, covering: needs_review, event_aggregates, onboarding, cover attribution, checkin events, notifications, ratings, news items, push tokens, password reset, vision quota, admin suspension/logs, friendships indexes, RLS tiers, manual cover media.

## 7. Shared Code & Cross-Component Dependencies

### `shared/` directory
- `shared/theme/tokens.js` — Design tokens (colors, spacing, radii, typography, shadow). Exported as ES module.
- `shared/styles/app.css` — Global CSS.
- **Mobile** watches this folder via `metro.config.js` (`config.watchFolders`).
- **Admin dashboard** does NOT currently import from `shared/` (uses Tailwind instead).
- **API** does NOT import from `shared/`.

### API ↔ Mobile contract
- All communication over HTTP REST to `EXPO_PUBLIC_API_BASE` (default `https://api.shelvesai.com`).
- Auth: `Authorization: Bearer <JWT>` header. Token stored in `expo-secure-store` (fallback: AsyncStorage).
- Token refresh: `POST /api/auth/refresh` called automatically when token is within 5 min of expiry.
- Mobile resolves API base at startup via env vars → `app.config.js` extra → `Constants.expoConfig.extra` → host URI fallback → Android emulator fallback.

### API ↔ Admin Dashboard contract
- All communication over HTTP to `/api/admin/*` (proxied by Vite dev server to `localhost:5001`).
- Auth: cookie-based (`admin_csrf` cookie) + CSRF token header (`X-CSRF-Token`) for mutations.
- Admin dashboard uses `axios` with `withCredentials: true`.

### Root `package.json`
- Not a workspace. Contains only `start`/`server` scripts that `cd api && node index.js`.
- Has stale dependencies (`mongodb-memory-server`, `supertest`) — likely leftover from earlier MongoDB era.

## 10. Do and Don't Patterns

### Do
- **Query layer separation:** All SQL lives in `api/database/queries/*.js`; controllers call query functions, not raw SQL. (`api/database/queries/shelves.js`, `api/database/queries/feed.js`)
- **Centralized error helpers:** `sendError(res, status, msg)` and `logError(context, err, metadata)` from `api/utils/errorHandler.js`.
- **Centralized constants:** Magic numbers (cache TTL, confidence thresholds, pagination limits) in `api/config/constants.js` with env-variable overrides.
- **Transaction helpers:** `transaction(fn)`, `transactionWithContext(userId, fn)` in `api/database/pg.js` — always use these for multi-step writes.
- **RLS context:** Use `queryWithContext(userId, sql, params)` for user-scoped queries so PostgreSQL RLS policies fire correctly.
- **Secure token storage:** JWT stored in `expo-secure-store` with AsyncStorage fallback (`mobile/src/services/api.js`).
- **HTTPS enforcement:** Mobile enforces HTTPS for API base in production (`enforceSecureApiBase`); admin dashboard enforces HTTPS for `VITE_API_URL` in production builds.
- **Rate limiting per endpoint:** Auth, registration, password reset, refresh, and admin login each have their own `express-rate-limit` instance with appropriate windows and maxes.
- **Adapter pattern for external APIs:** Each catalog source (TMDB, OpenLibrary, Hardcover) has its own adapter in `api/adapters/`.

### Don't
- **Don't bypass RLS:** Don't use raw `pool.query()` for user-scoped data — use `queryWithContext` or `transactionWithContext`.
- **Don't commit `.env` files:** `api/.env` is in `.gitignore`; it contains live credentials.
- **Don't use the root `package.json` for app code:** It is not a workspace; each component manages its own `node_modules`.
- **Don't assume `shared/` is available in admin-dashboard:** Metro watches it for mobile, but Vite does not — admin uses Tailwind for styling.
- **Don't use broad exception swallowing:** Error handlers log context + metadata before responding; errors are not silently dropped. (`api/utils/errorHandler.js`)
- **Don't use HTTP in production for mobile:** `enforceSecureApiBase()` throws and reports to Sentry if API base is HTTP in a production build.
